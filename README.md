## Project Requirements

### Building the Data Warehouse (Data Engineering)

#### Objective
Design and implement a data warehouse using SQL Server to integrate and organize banking transaction data for analytical and reporting purposes.

#### Specifications
- **Data Sources**: Import banking transaction data from multiple structured Excel files.
- **Data Quality**: Clean and standardize data, handle missing values, and remove duplicate records.
- **Integration**: Combine transactions, users, and cards data into a unified analytical model.
- **Scope**: Focus on processing the current dataset without historical version tracking.
- **Documentation**: Provide clear documentation describing the data warehouse schema and transformation process.

--- 

### BI : Analytics & Reporting (Data Analytics)

#### Objective
Develop SQL-based analysis and reporting to generate insights related to:
**Customer transaction behavior**
**Merchant and transaction performance**
**Transaction trends and financial activity patterns**

These insights support better financial monitoring and business decision-making.
